## 2025-05-14 - [Semantic Buttons and Dynamic ARIA Labels]
**Learning:** This application heavily relied on `div` elements for interactive components (Admin portal, Sync button) and lacked ARIA labels for icon-only utilities. Converting these to semantic `<button>` elements improves accessibility and keyboard navigation (Tab order) out of the box. Additionally, for toggle buttons (like mute/dark mode), `aria-label` must be updated dynamically in JavaScript to reflect the *action* that will be taken or the current state.
**Action:** Always audit for non-semantic interactive elements. When converting `div` to `button`, use `font-family: inherit` in CSS to prevent the browser's default button font from overriding the app's design. Ensure dynamic elements have their ARIA labels updated in the same functions that handle their visual state changes.
